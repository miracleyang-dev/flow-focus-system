# 心流 (Flow) - 个人效率系统

一个极简高效的个人专注力与效率管理应用。基于原生 Web 技术与 Node.js 构建，PWA 支持，助你快速进入"心流"状态。

## 核心特性

- **任务管理**：四象限优先级、拖拽排序、甘特图时间轴、循环任务
- **长期习惯**：时长/次数两种记录方式，可反复完成，支持编辑与 emoji 重新生成
- **水滴系统**：完成任务和习惯打卡获得水滴，量化专注成果
- **消费商城**：用水滴兑换自定义奖励，支持添加/编辑/删除商品
- **近期总结**：1-90 天自定义范围，基于任务/习惯/商城三大结构化日志，AI 生成洞察报告
- **AI 辅助**：任务分析拆解、总结洞察、emoji 自动生成
- **PWA**：可安装到桌面/移动端，离线可用

## 功能模块

| 模块 | 说明 |
|------|------|
| 仪表盘 | 每日概览、统计卡片、今日任务、常用链接、手动刷新按钮 |
| 倒空大脑 | 快速输入待办，支持 AI 分析拆解或手动添加 |
| 任务看板 | 四象限排列，桌面&移动端拖拽排序，标签筛选 |
| 甘特图 | 桌面14天横向 / 移动端5天垂直卡片 |
| 近期总结 | 自定义1-90天范围，汇总任务/习惯/消费数据，AI 洞察，下载报告 |
| 长期习惯 | 时长或次数记录，自定义水滴奖励，支持编辑修改 |
| 消费商城 | 水滴兑换奖励，自定义添加/编辑/删除商品 |
| 设置 | AI 配置、标签管理、常用链接、数据导入导出 |

## 水滴规则

完成任务：`floor(预估时间 / 60 * 优先级权重)`，权重：重要紧急 x4、重要 x3、紧急 x2、一般 x1

习惯打卡：时长类 `floor(小时 * 每单位水滴数)`（保底1滴），次数类 `次数 * 每单位水滴数`

## 数据架构

三个结构化日志统一管理，均保留最近 90 天：

| 日志 | 触发时机 | 记录内容 |
|------|---------|---------|
| `taskLogs` | 任务标记完成 | 名称、四象限、预估时长、获得水滴 |
| `habitLogs` | 习惯打卡 | 习惯ID、完成值、获得水滴 |
| `shopHistory` | 商品兑换 | 商品名称、图标、消费水滴 |

近期总结和 AI 洞察均基于以上三个日志生成，不受每日任务清理影响。

## 每日刷新机制

- 零点自动刷新：`scheduleMidnightRefresh()` 精确定时 + `visibilitychange` 兜底
- 手动刷新：仪表盘日期旁的刷新按钮
- 刷新内容：循环任务推进并重置、清除过期已完成非循环任务、清理超过90天的日志

## 技术栈

- **前端**: HTML5 + CSS3 + 原生 JavaScript（无框架）
- **后端**: Node.js 原生 `http` 模块
- **存储**: Redis（主） + JSON 文件备份 + localStorage 离线兜底
- **PWA**: Service Worker + manifest.json

## 快速开始

```bash
npm install
npm start
# 访问 http://localhost:3000
```

Redis 可选，无 Redis 时自动使用 JSON 文件存储。云端部署配置 `REDIS_URL` 和 `PORT` 环境变量即可。

## 后端与存储说明

- 健康检查：`GET /api/health`，返回服务端与 Redis 状态
- Redis 环境变量：`REDIS_URL` / `REDIS_PRIVATE_URL` / `REDIS_PUBLIC_URL`
- 本地备份：服务端同时写入 `data.json`
- 保存限制：单次保存请求体最大 5MB
